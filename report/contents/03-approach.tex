\section{Method}

We propose a reinforcement-learning (RL) based token routing framework with a shared memory bank to improve the effectiveness of linear attention when applying to a ViT for high resolution images.

\subsection{Overall Architecture}

Our study augments a standard ViT with a reinforcement-learning based token routing module and linear attention mechanism. Given an input image, the ViT patch embedding first converts the image into a sequence of $N$ patch tokens.
Instead of processing all tokens through every transformer layer, we introduce a lightweight policy network $\pi_\theta$ that dynamically routes tokens at selected stage transitions. The policy assigns each token an action, that is: \textit{keep}, \textit{remove}, \textit{store}, or \textit{retrieve}, allowing the model to filter redundant tokens and preserve informative ones.
Tokens assigned the \textit{keep} or \textit{retrieve} actions are packed into a shortened sequence of length $N_{\text{active}}$ and passed to the subsequent linear attention blocks. The attention output is computed as

\[
O = \phi(Q_{\text{active}})\big(\phi(K_{\text{active}})^T V_{\text{active}}\big).
\]

We aim to make $N_{\text{active}} \ll N$, the effective computational cost is reduced while maintaining the model's ability to focus on important tokens. Meanwhile, contextual information from discarded or stored tokens can be preserved through the shared memory mechanism, allowing global context to bypass the linear attention bottleneck.

\subsection{RL-Based Token Routing}

To tackle the limitations of linear attention, we introduce a lightweight
RL policy network that dynamically routes tokens
during inference. Given a token representation $x_i \in \mathbb{R}^d$, the policy network
$\pi_\theta$ computes an action distribution

\[
\pi_\theta(a_i \mid x_i) = \text{Softmax}(W_2 \sigma(W_1 x_i)),
\]
where $W_1$ and $W_2$ are learnable parameters and $\sigma(\cdot)$ denotes a
nonlinear activation function. The policy selects an action

\[
a_i \in \{\text{keep}, \text{remove}, \text{store}, \text{retrieve}\}.
\]

Each action modifies the token flow as follows:

\begin{itemize}
\item \textbf{Keep}: the token remains in the active sequence and participates
in linear attention.
\item \textbf{Remove}: the token is discarded to reduce sequence length.
\item \textbf{Store}: the token is compressed into a shared memory bank.
\item \textbf{Retrieve}: the token receives contextual information from the
memory bank.
\end{itemize}

Let $\mathcal{A}$ denote the set of tokens selected by the policy to remain
active. Linear attention is then computed only over this subset, which reduces the effective token count $N_{\text{active}} = |\mathcal{A}|$.
The policy is trained with reinforcement learning to maximize a reward that
balances model performance and computational efficiency.

\subsection{Shared Memory Bank}

Tokens assigned to the \textit{store} action are aggregated into a shared memory representation:

\[
B = \frac{1}{|\mathcal{S}|} \sum_{j \in \mathcal{S}} x_j,
\]
where $\mathcal{S}$ denotes the set of stored tokens.

Tokens requesting contextual information retrieve it through a residual update

\[
\hat{x}_i = x_i + W_r B,
\]
where $W_r$ is a learnable projection matrix.

This memory pathway allows global contextual information to bypass the linear attention bottleneck while keeping the computational cost low.

\subsection{Training Objective}

The routing policy is trained using reinforcement learning. The reward function balances two objectives: preserving downstream task performance, minimizing the number of active tokens. The reward is defined as
\[
R = \alpha \, \text{Acc} - \beta \frac{N_{\text{active}}}{N},
\]
where $\text{Acc}$ represents task performance, and $\alpha,\beta$ control the trade-off between accuracy and efficiency.
